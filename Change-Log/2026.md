# Release Notes 2026

## Date 12/02/2026 
## Release 1.2602.5.0
 
 
## [NEW]
###	POS
		- Display client's fidelity card barcode when buying a voucher (StoreSetup parameter PosShowFidelityCardCodeWhileBuyingVoucher needs to be true).
 
 
###	BO - Contacts
		- Added a new form for adding points in which you can specify the number of points, for which store, and a note.
 
 
###	BO - Stock, Causes, Warehouses 
		- Added Warehouse Types viewer in BackOffice (needs permission sbWarehouseTypes).
		- Permit the user to view and modify the Store's Warehouse Types on Stores editor (if Warehouse Management is enabled through StoreSetup parameter IsWarehouseManagementEnabled).
 
 
##	[FIXES]
###	POS
		- Fixed an issue where discount causes would not take into account if the discount was fixed valoric discount.
		- Fixed a bug when the user pressed the (+) button to see the discounts the app would show the product details.
		- Fixed a rounding issue where the total % discount was calculated with a different formula than the row % discount, resulting in small differences.
		- Fixed an issue when the value of the sale was greater than the return products the user could not finalize the sale without the permission of allowing cash returns.
		- Fixed an issue when resuming a suspended receipt containing products with no barcode, the app would freeze.
		- Fixed the calculation of the total % discount when doing a mixed transaction - no longer take into account return products.
 
 
###	BO - Entrance Goods 
		- Fixed a divided by 0 error on the grid.
 
 
###	BO
		- Made Warehouse Type column on Movements Form and Stock column on Outgoing Document Details uneditable.
 
 
##	[IMPROVEMENTS]
###	BO - Contacts
		- Now contacts that have set for their fidelity card type a maximum points limit, it will be shown, as well as how much points are still needed to reach the limit.			
 
 
###	BO - Delivery Documents print report
		-
 
 
###	BO - Stock, Causes, Warehouses
		- Improved the viewing of Movements on Products Form in order to show all product status' stock (Available, Reserved, Transit, Defective and Wrecked) as well as the movement's Warehouse Type.
		- Added Warehouse Type on Lots Analisys.
 
 
###	BO - Labels
		- Now creates labels for all size ranges on the scanned product barcode.
 
 
###	POS - Licences
		- Added a new parameter that will allow the user to see in the POS the number of available licenses and the number of used licenses after logging in. The parameter is LicensesPriorExprirationWarningDays (default: 0).
 
 
###	POS - Omaggi
		- Fixed an issue where the secondary cash registers could not see the stock to use Omaggi.
 
 
###	POS - Discounts
		- Added auto expand of tickets' discounts (StoreSetup parameter POSAutoExpandTicketRowDiscounts needs to be true).
 
 
###	POS - Print receipt
		- Added functionality to limit the number of characters printed on the receipt for the product name (StoreSetup parameter POSPrintTicketMaxProductNameChars will set the max number of characters).
 
 
###	Sync
		- Improved synchronization of Warehouse Types and Store Warehouses.
		- Improved synchronization of Contacts.

## Date 09/02/2026
## Release 1.2602.3.0   

### [NEW]
- Nessuna nuova funzionalità introdotta in questa release.

### [FIXES]
#### POS
- **Self Invoice Management**: Added setup parameter `PosSaleAllowSelfInvoice` that verifies if a self invoice contact pays with `NOT_PAID` payment method and limits any other payment method for this contact.
- **Warehouse Returns**: Fixed an issue where trying to return a product with negative or 0 stock would infinitely display a message that the stock available is too little and blocked the app.

#### BO (BackOffice)
- **DDT Printing**: DDT print now displays the quantity values as integers.
- **Fidelity Reporting**: Fixed columns regarding transactions with fidelity for the report "RMV Totale".

### [IMPROVEMENTS]
- **Warehouse Logic**: Improved the logic of warehouse assignment for sales and returns based on `StoreWarehouses`.
- **Product Movements**: Improved displaying of movements based on stock status (Available/Reserved/Defective/Transit/Wrecked).
- **Sync**: Improved sync of warehouse types and store warehouses.

## Date 16/01/2026  
## Release 1.2601.3.0

---

## [NEW]

### Movements - Causes - Stock
- Added new stock state, which manages completely wrecked products stock.

### WEB
- Added BuyBook web application that allows creating and editing products linked to a BuyerPlan.
- Products created or modified in BuyBook are immediately visible in IPOS.

### POS
- Added the possibility to reprint the store credit barcode (voucher code) when reprinting a return receipt.

---

## [IMPROVEMENTS]

### Vouchers
- Removed **"HYBRID"** Isolation Level from Voucher Generators.
- Added new property **"Can Issue on Local DB"**, which specifies if the voucher can be issued without having connection to the Central HQ Server in order to define the true Isolation Level of the Voucher.

### POS
- The Reduced Prices as promotion now displays the percentage of the discount applied besides the value (visually only, not saved).
- The search for receipts can now be done by receipt number, besides the date of issue.  
  This is available only if the receipt number does not reset daily  
  (app parameter: `ResetReceiptNumberDaily = False`).
- StoreDayClose improvement.

### BackOffice
- In the Promotions Management section, the brand of products is now displayed, only if the store has `storeBrand` set on at least one store.
- Receipt import improved to recognize products also by product code.
- Optimized discount handling during receipt import: movement discounts are no longer created when no discount is present.
- Changed Zebra Labels print default extension to `.zpl`.
- Added a new button on the **BuyerPlanDetailsForm** with the standard **Generate Labels** functionality.
- Improved the query for viewing **Product Selector – Price Lists** tab.
- Improvement on Incoming Document **Generate Labels**.

---

## [FIXES]

### Promotions with Maximum Season Start Date
- Fixed an issue where the Max Season Start Date was also being included as a trigger when applying a promotion.  
  Now the promotion applies only for dates **before** the selected date.
- Fixed an issue where disabled manual movement causes were still proposed during manual movements.
- Fixed movements import from Headquarter to Stores where `CreateStoreId` was not populated, causing synchronization issues.  
  `CreateStoreId` is now automatically set to `1` when no value is provided in the import procedure.
- Removed the association between the Order and Receipt on XPO.
- Changes for THUN File Manager
